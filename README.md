## Abstract

This project analyses the **HAM10000** dermatoscopic dataset (10,015 images, 7 diagnostic classes) to support early melanoma detection. We combine **statistical hypothesis testing** on patient metadata with **handcrafted image features** and train a family of tabular machine-learning models. The best tuned LightGBM achieves **Accuracy 0.807**, **Macro-F1 0.562**, and **OvR ROC-AUC 0.946**; a dedicated two-tier pipeline further improves melanoma recall from 0.54 to a clinically more useful operating point. The dataset is severely imbalanced (66.9% melanocytic nevi), so macro-averaged and per-class metrics drive all comparisons.

## Research questions

1. Are demographic and clinical features significantly associated with the lesion type?
2. Can a tabular classifier built on metadata and handcrafted image features achieve acceptable multi-class performance?
3. Does combining metadata with image features outperform either feature set alone?

## Headline results

| Experiment                          | Accuracy | Macro-F1 | Weighted-F1 | ROC-AUC (OvR) |
|-------------------------------------|---------:|---------:|------------:|---------------:|
| Metadata only                       | 0.714 | 0.401 | 0.737 | 0.875 |
| Image features only                 | 0.755 | 0.428 | 0.758 | 0.901 |
| **Metadata + Image (combined)**     | **0.811** | **0.563** | **0.806** | **0.946** |
| XGBoost (combined)                  | 0.815 | 0.578 | 0.815 | 0.948 |
| LightGBM (combined)                 | 0.817 | 0.575 | 0.813 | 0.942 |
| **Tuned LightGBM (best model)**     | **0.807** | **0.562** | **0.814** | **0.946** |

Melanoma-focused analysis (baseline vs. two-tier pipeline) is reported in `Report/Report.ipynb`, Section 11.

## How to reproduce

1. **Clone / open** this folder in Jupyter Lab or VS Code.
2. **Create environment** (Python ≥ 3.10):
   ```bash
   python -m venv .venv
   .venv\Scripts\activate            # PowerShell: .venv\Scripts\Activate.ps1
   pip install -r requirements.txt
   ```
3. **Download the dataset** following `data/README.md` (Kaggle CLI or Harvard Dataverse).
4. **Run the pipeline either from the command line or from a notebook:**

   *Command line* (uses the modules under `src/`):
   ```bash
   python -m scripts.run_eda                 # EDA + hypothesis tests (§2–3)
   python -m scripts.extract_features        # caches results/image_features.csv (§4)
   python -m scripts.train_baselines         # six baselines, saves comparison plots (§6)
   python -m scripts.tune_lightgbm           # randomised search + final tuned model (§9)
   python -m scripts.run_melanoma_pipeline   # threshold sweep + binary gate + two-tier (§11)
   # Or run everything in order:
   python -m scripts.run_full_pipeline
   ```

   *Notebooks* — open the deliverables in this order:
   - `Proposal/Proposal.ipynb`   (read-only narrative)
   - `Milestone/Milestone.ipynb` (loads cached `results/results_summary.json`)
   - `Report/Report.ipynb`       (full pipeline — expect ~15–25 min on CPU; image-feature extraction is the bottleneck)

5. **Re-export HTMLs** if needed:
   ```bash
   jupyter nbconvert --to html Proposal/Proposal.ipynb  Milestone/Milestone.ipynb  Report/Report.ipynb
   ```

### Using the `src/` package from your own notebook

```python
from src.config import resolve_paths
from src.data import load_metadata, build_image_index
from src.features import load_or_build_features
from src.preprocessing import build_training_matrices
from src.models import train_all_baselines
from src.plotting import results_to_dataframe, plot_model_comparison

paths = resolve_paths()
df = load_metadata(paths["metadata_path"])
img_paths = build_image_index(paths["base_dir"])
features = load_or_build_features(df, img_paths,
                                  cache_path=f"{paths['output_dir']}/image_features.csv")
td = build_training_matrices(df, features)
results = train_all_baselines(td)
comp = results_to_dataframe(results)
plot_model_comparison(comp, paths["output_dir"])
```
